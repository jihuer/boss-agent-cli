# 多平台适配器调研索引

本目录存放面向 Issue #90（拉勾 / 智联 / 猎聘 多平台适配器）的**公开接口调研报告**。

## 约束

- 只做**公开接口**分析（浏览器 DevTools、官方 API 文档、开源项目可观察的部分）
- **不逆向**加固代码、不解析混淆 JS、不发布绕过方案
- 目标是评估「是否适合接入 boss-agent-cli 的 Platform 接口抽象」，不是写 crawler

## 交付物规范

每个平台一份 `docs/research/platforms/<name>.md`，按以下 7 项清单交付：

1. 登录链路（扫码 / 账号密码 / 手机验证码 / stoken 或 csrf 类防爬字段）
2. 核心 API 端点清单（对标 BOSS 直聘的 7 个基础端点）
3. 请求签名 / 防爬机制（Cookie、Header、Body、指纹）
4. 响应结构（JSON 包络、错误码语义）
5. CDP / Browser 通道可行性（是否可复用现有 `BrowserSession` 模式）
6. 风控触发阈值（多少次请求会触发滑块 / 验证）
7. 与 BOSS 直聘的协议差异矩阵（支持 / 改造 / 不支持）

## 三家平台对比速览

| 维度 | 拉勾 | 智联 | 猎聘 |
|------|------|------|------|
| 登录难度 | 低（可 httpx） | 高（极验滑块） | 高（点选验证码） |
| API 数量 | 9 | 10 | 10 |
| Token 有效期 | 24h | 7 天 | 30 天 + 2h 短 |
| 签名机制 | JWT + AES | CSRF + 新版签名 | 双 token + MD5 |
| 风控严度 | 严 | 中 | 搜索宽松/写操作极严 |
| 接入成本 | 3-4 周 | **2-3 周** | 4-5 周 |
| 用户画像匹配 | 互联网偏重 | **均衡** | 高端错位 |
| **最终建议** | ❌ | ✅ | ❌ |

## 认领流程（其他平台）

1. 在 Issue #90 下评论「我认领 XX 平台调研」
2. 基于 [lagou.md](lagou.md) 的模板产出 `docs/research/platforms/<name>.md`
3. 通过 PR 提交调研报告
4. 所有已认领平台都产出结论后，再讨论 Platform 接口抽象的实施 PR

> 调研结论如果是「不建议接入」，同样有价值——帮项目省掉一次无意义的大重构。
